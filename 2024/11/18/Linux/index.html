Linux发行版

提供了内核+系统级程序的完整封装，称之为Linux发行版。

​ 不同发行版基本命令100%相同，部分操作不同（软件安装）。

本课程使用CentOS辅以Ubuntu进行讲解。

WSL（Windows Subsystem for Linux）

​ 用于Windows系统之上的Linux子系统。

​ 可以无需单独虚拟一套硬件设备，直接使用主机的物理硬件，不影响Windows系统本身

基于WSL得到Ubuntu发行版环境，然后对比不同Linux发行版之间命令的差异（基础命令100%相同）

可以跳过不学习。

WSL作为Window 10系统带来的全新特性，逐步颠覆开发人员既有的选择：

1. 传统获取Linux操作系统环境，是安装完整的虚拟机，如VMware
1. 使用WSL，可以非常轻量化的方式，得到Linux系统环境

1.windows11开启wsl2并安装Ubuntu 20.04_windows开启wsl2-CSDN博客

管理员身份打开cmd：C:\Windows\System32>wsl –set-default-version 1

Linux命令基础

命令行：即Linux终端，可以提供字符化的操作页面供命令执行。

命令：即Linux程序。一个命令就是一个Linux的程序，即linux操作指令，是系统内置的程序，可以以字符化的形式使用。

命令通用格式：

ls命令

ls命令的作用是列出目录下的内容，语法细节如下：

  • -a -l -h 是可选的选项
  • linux路径是此命令可选的参数
  • 当不使用选项和参数，直接使用ls命令本体，表示：以平铺形式，列出当前工作目录（默认home目录）下的内容（文件文件夹）。

home目录和工作目录

​ home目录：每个Linux操作用户在Linux系统的个人账户目录，路径在：/home/用户名

​ 当前工作目录：Linux命令行在指向命令的时候，需要一个工作目录，打开命令行程序（终端）默认设置工作目录在用户的home目录。

ls命令的选项

命令的选项是可以组合使用的

eg. ls -lah（顺序可调换） 等同于 ls -a -l -h

cd命令

英文：Change Directory

功能：切换当前工作目录

语法：cd [Linux路径]

  • 没有选项，只有参数，表示目标路径
  • 使用参数，切换到指定路径
  • 不使用参数，切换工作目录到当前用户的HOME

pwd命令

英文：Print Work Directory

作用：输出当前所在的工作目录

- pwd命令，没有选项，没有参数，直接使用即可

练习：通过cd和ls找games目录

  1. /var/lib/games
  2. /var/games

​

相对路径，绝对路径和特殊路径符

绝对路径：以根目录为起点，描述路径的一种写法，路径描述以 / 开头

相对路径：以当前目录为起点，描述路径的一种写法，路径描述无需以 / 开头

特殊路径符

  • . 表示当前目录；比如：cd . 或 cd ./Desktop
  • .. 表示上一级目录；比如：cd .. 或者 cd ../..
  • ~ 表示HOME目录，即 /home/xp，比如：cd ~ 或 cd ~/Desktop
ctrl + l 清屏

mkdir命令

功能：用来创建新的目录（文件夹）

英文：Make Directory

语法：mkdir [-p] [Linux路径]

-

  • -p的作用：可选，表示自动创建不存在的父目录，适用于创建连续多层次的目录
练习

​ mkdir -p ~/itcast/itheima/nice/666

文件操作命令

touch命令

  1. 可以通过颜色区分文件与文件夹

  2. ls -l 最开头为 d 文件夹， - 文件

功能：用于创建一个新的文件

语法：touch Linux路径

  • 参数必填，表示要创建的目录的路径，相对、绝对、特殊路径符都可以使用

cat命令

功能：用于查看文件内容，全部展示

语法：cat Linux路径

  • 参数必填，表示要创建的目录的路径，相对、绝对、特殊路径符都可以使用

more命令

功能：用于查看文件内容，可翻页查看

语法：more Linux路径

  • 参数必填，表示要创建的目录的路径，相对、绝对、特殊路径符都可以使用

  • 使用空格进行翻页，使用q推出查看

mv当参数2不存在时，改名

cp命令

功能：用于复制文件/文件夹

语法：cp [-r] 参数1 参数2

  • -r选项，可选，用于复制文件夹使用，表示递归

  • 参数1：Linux路径，表示被复制的文件/文件夹

  • 参数2：Linux路径，表示要复制的文件/文件夹

mv命令

功能：用于移动文件内容

语法：mv 参数1 参数2

  • 参数1，Linux路径，表示被移动的文件/文件夹
  • 参数2，Linux路径，表示要移动去的地方，如果目标不存在，则进行改名，确保目标存在

rm命令

功能：用于复制文件/文件夹

语法：rm [-r -f] 参数1 … 参数n

  • -r选项，可选，文件夹删除
  • -f选项，可选，用于强制删除（不提示，一般用于root用户 su - root）
  • 参数，表示被删除的文件/文件夹路径，支持多个，空格隔开
  • 参数也支持通配符 * ，用以做模糊匹配 test * test * test

which命令

功能：查找命令的程序文件

语法：which 要查找的命令

  • 无需选项，只需要参数表示查找哪个命令

find命令

功能：用于查找指定的文件

语法：

  • 按文件名查找：find 起始路径 -name “被查找文件名”
    • 支持通配符
  • 按文件大小查找：find 起始路径 -size +|-[kMG]
    • +|-代表大于|小于

grep命令

功能：从文件中通过关键字过滤文件行

语法：grep [-n] 关键字 文件路径

  • 选项-n，可选，表示在结果中显示匹配的行的行数
  • 参数，关键字，必填，表示过滤的关键字，建议使用””将关键字包围起来
  • 参数，文件路径，必填，表示要过滤内容的文件路径，可作为管道符的输入

wc命令

功能：统计文件的行数，单词数量，字节数，字符数等

语法：wc [-c -m -l -w] 文件路径

  • 不带选项默认统计：行数，单词数，字节数
  • -c字节数，-m字符数，-l行数，-w单词数
  • 参数，被统计的文件路径，可作为管道符的输入

管道符 |

将管道符左边命令的结果，作为右边命令的输入

练习

统计文件中带有itcast关键字的有几行

cat Desktop/test.txt | grep itcast | wc -l

统计文件中带有itheima关键字的结果中有多少个单词

cat Desktop/test.txt | grep itheima | wc -w

echo命令

​ 功能：在命令行内输出指定内容

​ 语法：echo 输出的内容

  • 无需选项，只有一个参数，表示要输出的内容，复杂内容用 **””**包围

`反引号符

​ 被 ` 包围的内容，会被作为命令执行，而非普通字符

重定向符

  1. 一个大于，将左侧命令的结果，覆盖写入到符号右侧指定的文件中
  2. 两个大于，将左侧命令的结果，追加写入到符号右侧指定的文件中

tail命令

​ 功能：查看文件尾部内容，并可以持续跟踪

​ 语法：tail [-f -num] Linux路径

    - -f选项：持续跟踪，-num：启动的时候查看尾部多少行，默认10
    - Linux路径：表示被查看的文件
练习

​ echo “我当前的工作目录是pwd“ > work.txt

 echo "内容" >> work.txt

​ tail -f work.txt

vi/vim编辑器

​ 功能：命令行模式下的文本编辑器，用来编辑文件

​ vim是vi的升级版，一般用vim即可，包含全部vi功能

​ 语法：vi 文件路径 / vim 文件路径

​ 运行模式：

    - 命令模式：默认的模式，可以通过键盘快捷键控制文件内容
    - 输入模式：通过命令模式进入，可以输入内容进行编辑，按Esc退出命令模式
    - 底线命令模式：通过命令模式进入，可以对文件进行保存、关闭等操作
